# DEMO_SCENARIOS — сценарии демонстрации

Перед демо удобно загрузить примерные данные:

```bash
curl -X POST http://localhost:8000/demo/seed
```

(в UI — кнопка «Загрузить демо»). Все запросы можно слать через вкладку **Chat**
или через `POST /chat` / специализированные эндпоинты.

---

## Демо 1. Большая задача → чеклист → календарь

**Запрос в чате:**

> Мне нужно за следующую неделю подготовить презентацию по проекту. Разбей на
> шаги и запланируй.

**Ожидаемое поведение:**

1. Supervisor → **Planning Agent**.
2. Цель разбивается на задачи (`split_goal_into_tasks`): собрать материалы,
   сделать структуру, подготовить слайды, прогнать демо, отрепетировать.
3. Оценка длительности (`estimate_task_duration`).
4. Поиск свободных окон и распределение по дням (`schedule_tasks_to_free_slots`)
   — создаются события календаря, связанные с задачами.
5. **Verifier** подтверждает, что задачи и события реально в storage.
6. **Reflection** проверяет реалистичность (конфликты, дедлайны).

**Что показать на защите:** чат с ответом-планом; вкладку **Tasks** (созданные
задачи/чеклист); вкладку **Calendar** (события по дням); раскрытые **tool calls**
в чате; блоки **Reflection** и **Verifier**; при желании `logs/agent.log`.

Эквивалент через API: `POST /plan {"text":"...","horizon_days":7}`.

## Демо 2. «Что мне делать дальше?»

**Запрос в чате:**

> Что мне сейчас делать дальше?

**Ожидаемое поведение:**

1. Supervisor → **Prioritization Agent**.
2. Смотрит задачи, дедлайны, календарь, просрочки и память.
3. Выбирает главную задачу, объясняет выбор, предлагает 2–3 альтернативы,
   называет риски; переводит выбранную задачу в `in_progress`.

Пример ответа: «Сейчас лучше заняться "Починить баг на проде" — приоритет
urgent, просрочена, уже в работе…».

**Что показать:** список задач до запроса (вкладка Tasks); рекомендацию и
объяснение в чате; изменение статуса задачи на `in_progress`; tool calls.

Эквивалент через API: `POST /prioritize`.

## Демо 3. Вечерний дайджест + Сонная фея

**Запрос в чате:**

> Закрой мой день и переведи меня в режим сна.

**Ожидаемое поведение:**

1. Supervisor → **Sleep Fairy Agent**.
2. Формируется вечерний дайджест (`generate_evening_digest`): что сделано/не
   сделано, события, просрочки, план на завтра.
3. Незавершённые/просроченные задачи переносятся на завтра
   (`reschedule_overdue_tasks`).
4. Мягкий план на утро + сообщение, что критичного на сегодня не осталось +
   короткий ритуал завершения дня.
5. **Verifier** проверяет: дайджест сохранён, задачи обновлены.
6. **Reflection** проверяет, что незавершённое обработано и нет конфликтов на завтра.

**Что показать:** ответ «Сонной феи» в чате; перенос задач (вкладка Tasks);
вкладку **Digest** (сохранённый вечерний дайджест + история); файл
`data/digests.json`.

Эквивалент через API: `POST /sleep-mode`.

## Дополнительные быстрые проверки

- Память: «Запомни, что я лучше работаю по утрам» → вкладка **Memory**.
- Задача: «Создай задачу купить билеты к пятнице» → вкладка **Tasks**.
- Календарь: «Создай событие созвон с командой завтра в 16:00» → **Calendar**.
- Дайджест: «Сделай утренний дайджест» → вкладка **Digest**.
- Persistence: создайте задачу, затем `make down && make up` — задача на месте.
