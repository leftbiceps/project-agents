# METRICS — метрики качества

Метрики считает скрипт `backend/evals/eval.py`: он прогоняет набор сценариев
через оркестратор и сравнивает результат с ожиданиями (агент, инструмент,
пост-условие в storage).

## Запуск

```bash
cd backend
python evals/eval.py
```

Скрипт печатает таблицу по сценариям и сводные метрики (плюс JSON). Работает в
любом режиме LLM; в rule-based режиме проверяется детерминированный путь.

## Определения метрик

| Метрика | Что измеряет | Как считается в eval |
|---------|--------------|----------------------|
| **Task Success Rate** | доля запросов, где цель достигнута | пост-условие сценария истинно (нужный объект появился/изменился в storage) |
| **Tool Call Accuracy** | доля корректных вызовов инструментов | ожидаемый инструмент присутствует и все вызовы завершились `ok` |
| **Routing Accuracy** | правильность маршрутизации Supervisor | `agent == ожидаемый агент` |
| **Planning Quality** | разумность разбиения и распределения | (качественно) число задач/событий и отсутствие предупреждений в сценарии планирования |
| **Calendar Conflict Rate** | доля планов с конфликтами (меньше — лучше) | число конфликтующих событий, делённое на число сценариев |
| **Memory Usage Accuracy** | корректность работы с памятью | факт сохранён и находится поиском (сценарий памяти + использование в приоритизации) |
| **Checklist Completion Accuracy** | корректность чеклистов | чеклист создан, связан с задачей, прогресс считается (сценарий разбивки) |
| **Autonomy Rate** | сколько шагов без лишних уточнений | доля сценариев с непустым ответом без ошибки |
| **Human Confirmation Precision** | подтверждение только для критичных действий | Reflection флагует `delete_*`, но не флагует обычное создание |
| **Digest Usefulness** | полнота дайджеста | доля ожидаемых разделов (задачи, события, рекомендация, план) в тексте |
| **Verification Pass Rate** | доля сценариев, где Verifier подтвердил изменения | `verification.passed` среди сценариев с изменениями |

## Базовые значения (rule-based режим, без LLM)

Прогон `python evals/eval.py` на демо-данных:

| Метрика | Значение |
|---------|----------|
| Routing Accuracy | 1.00 |
| Tool Call Accuracy | 1.00 |
| Task Success Rate | 1.00 |
| Autonomy Rate | 1.00 |
| Verification Pass Rate | 1.00 |
| Human Confirmation Precision | 1.00 |
| Digest Usefulness | 0.75 |
| Calendar Conflict Rate | 0.25 |

Примечания: `Digest Usefulness` < 1.0, т.к. вечерний/«сонный» дайджест содержит
не все искомые маркеры разделов. `Calendar Conflict Rate` > 0 возникает, когда
вручную созданное событие пересекается с авто-запланированным блоком — такие
случаи Reflection помечает как требующие внимания. В LLM-режиме значения могут
отличаться (зависит от модели); скрипт одинаково применим для сравнения моделей.

## Как расширять

Добавьте кортеж в `SCENARIOS` (`имя, сообщение, ожидаемый агент, ожидаемый
инструмент|None, проверка(storage, ответ)->bool`). Метрики пересчитаются
автоматически.
